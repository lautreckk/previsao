Find the Conta tab win rate display (currently at line 366):
```tsx
{user.total_predictions > 0 ? Math.round((user.total_wins / user.total_predictions) * 100) : 0}% acerto
```

Change to use `total_wins / (total_wins + total_losses)` to match the Resumo tab formula. This excludes pending bets from the denominator:
```tsx
{(user.total_wins + user.total_losses) > 0 ? Math.round((user.total_wins / (user.total_wins + user.total_losses)) * 100) : 0}% acerto
```

WHY: `total_predictions` includes pending (unresolved) bets in the denominator, which deflates win rate. The Resumo tab already uses `won / (won + lost)`, so this makes both tabs consistent.
  </action>
