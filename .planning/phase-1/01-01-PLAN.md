<task type="auto">
  <name>Task 2: Create backfill migration script for historical camera predictions</name>
  <files>scripts/backfill-camera-stats.mjs</files>
  <action>
Create a Node.js script that fixes historical user stats by counting all existing camera_predictions and adding to users table totals. This script runs AFTER Task 1 is deployed (so new bets are tracked correctly going forward).

The script must:
1. Import `createClient` from `@supabase/supabase-js` using the `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` env vars (read from `.env.local` using dotenv or inline).
2. Query all camera_predictions grouped by user_id: `SELECT user_id, COUNT(*) as bet_count, SUM(amount_brl) as total_wagered FROM camera_predictions GROUP BY user_id`.
3. For each user, fetch their current `total_predictions` and `total_wagered` from the users table.
4. Add the camera counts to the existing values (do NOT replace — other bet types are already counted).
5. Recalculate `level` using the same formula as the API.
6. Update the users table.
7. Log progress: `Updated user {user_id}: +{bet_count} predictions, +{total_wagered} wagered (new total: {new_total})`.
8. At the end, log summary: `Backfill complete: {N} users updated, {M} total camera predictions counted`.

Use Supabase RPC or raw SQL via `.rpc()` for the aggregation query. If RPC is not available, paginate through camera_predictions manually.

Add a DRY_RUN mode (default true) that logs what WOULD change without writing. Set `const DRY_RUN = process.argv.includes('--execute');` inverted so `node scripts/backfill-camera-stats.mjs` shows dry run and `node scripts/backfill-camera-stats.mjs --execute` actually writes.

Include shebang and usage comment at top:
```javascript

//        node scripts/backfill-camera-stats.mjs --execute  (apply changes)
```
